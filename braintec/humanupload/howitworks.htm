Humanupload Program
 Monitoring the whole human thought at once is an interesting but difficult task. There's so much data to analyze that it almost seems impossible! Fortunately, the data analysis task can be easily broken up into little pieces that can all be worked on separately and in parallel. None of the pieces depends on the other pieces. The millions of MEMES that form the first humanupload are divided into 0.25 Mbyte chunks which we call "work-units". These are sent from the Braintec server over the Internet to volunteering computers around the world to analyze. There is some additional info added to each chunk of data, so the total of one work-unit comes out to about 340 kbytes. Your computer will do between 2.4 trillion and 3.8 trillion mathematical operations (flops or floating point operations in technical jargon) to complete one work-unit.

Braintec hopes to convince you to allow us to borrow your computer when you aren't using it and to help us unravelling the mystery of human thought. You simply download a free software program called the Humanupload Program. This program captures the otherwise wasted processor cycles of your computer and applies them to analyze the first humanupload. It works like a special kind of screensaver that can go get a chunk of data from us over the Internet, analyze that data, and then report the results back to us. When you need your computer back, the Humanupload Program instantly and automatically gets out of the way and only continues it's analysis when you are finished with your work.

The Humanupload Program actually consists of two parts: an application and a screensaver. The application does all the work:  it downloads a work-unit through the Internet, analysis it, returns the results and gets another work-unit. The screensaver program runs when you haven't used your keyboard or mouse for a while.  It displays color graphics showing the data analysis process.

Braintec servers
 The main Braintec server consists of three Sun Enterprise 650 series computers. One holds the user database, containing information on each of the BraintecGrid volunteers. The user database also holds information on the amount of work done by each type of microprocessor architecture and by each operating system to which the BraintecGrid has been ported.

The second server system holds the science database in an ever-expanding array of redundant disks (currently a 128-Tbyte RAID 0+1).

The third server system contains the work unit storage, handling distribution of work units and storage of returned results. Communications between clients and the server use the hypertext transfer protocol (HTTP). We chose this protocol because many Internet volunteers might be behind firewalls that prohibit most traffic but permit access to the World Wide Web.
